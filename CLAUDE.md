## Форматы CSV

### data/rating.csv
```
№,ФИО,Пин,Посещение,Семинары,Тесты,Контрольная,Общий рейтинг
1,Иванов Иван Иванович,1001,85,90,78,7,84.5
```
- Пин — код который преподаватель сообщает студентам лично
- Контрольная — кол-во правильных ответов из 10
- Общий рейтинг = 0.1*посещение + 0.25*семинары + 0.25*тесты + 0.4*(контрольная/10*100)

### data/topics.csv
```
№,Название темы,Краткое описание,Лекция,Семинар,Тест,Видимость
1,Введение,Источники права,https://ahaslides.com/XXX,https://...seminar1/,https://ahaslides.com/YYY,да
```
- Пустой URL -> кнопка disabled
- Видимость "нет" -> тема скрыта

### data/library.csv
```
Тип,Название,Автор,Год,Файл,Примечание
Учебник,Каноническое право,Цыпин В.,2009,tsypin.pdf,Основной
Словарь,Православная энциклопедия,,2025,,Скоро
```
- Пустое поле Файл -> бейдж "Скоро"
- Типы: Учебник / Хрестоматия / Пособие / Словарь

## Система оценивания (шаблон)

| Компонент | Вес | Условие |
|---|---|---|
| Посещение | 10% | — |
| Семинары | 25% | — |
| Тесты | 25% | — |
| Контрольная | 40% | >= 6 из 10 |
| Итог | 100% | >= 61% И контрольная >= 6/10 |

Контрольная: письменно на бланке, 10 случайных вопросов из тестов всех тем, без вариантов.
Отработка семинаров: рукописный ответ -> фото -> PDF -> LMS
Отработка тестов/контрольной: один раз на консультации (суббота 15:10-16:40, F305)

## КРИТИЧЕСКИЕ ОШИБКИ

### Cloudflare ломает JS (email в HTML)
Симптом: "load is not defined", страница виснет
Правило: НИКОГДА не писать email в HTML-файлах
```bash
grep -n "cdn-cgi\|cfasync" файл.html
sed -i 's|<script data-cfasync="false" src="/cdn-cgi/...email-decode.min.js"></script><script>|<script>|g' файл.html
```

### JS навигации вне тега script
Симптом: белый экран / страница не открывается
Проверка: весь nav JS должен быть внутри <script>...</script>
```bash
tail -15 файл.html  # последняя script-закрывашка должна быть ДО </body>
```

### Файл обрезан
```bash
grep -n "</body>\|</html>" файл.html | tail -3
```

### CSV 404
Симптом: "SyntaxError: Unexpected token <"
Решение: проверить что data/*.csv загружены в репозиторий

### Страница виснет в LMS
Причина: относительный URL
Решение: заменить на абсолютный https://julian90807-rgb.github.io/[slug]/...

## Алгоритм нового курса

1. Создать репозиторий [slug] на GitHub
2. Settings -> Pages -> main branch, /root
3. Скопировать файлы из canonical-law как шаблон
4. Заменить: название, slug в URL, темы, литературу
5. Заполнить data/*.csv
6. Создать glossary/index.html и library/index.html
7. Создать materials/seminar1/ ... seminarN/
8. Обновить nav URL во всех файлах
9. Добавить URL-элементы в LMS
